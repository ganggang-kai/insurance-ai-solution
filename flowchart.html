N04 — 关注状态查询（跳转节点）

节点类型跳转节点
切换规则按时间（每5秒触发一次）
静默时间5 秒
重复次数12 次（总计 60 秒等待窗口）
动作调用「查询关注状态 API」
已关注 →跳出循环 → N05 专属码交付
60s 未关注 →跳出循环 → 短信直链（降级通道）
循环中→ PROMPT 普通节点「点关注按钮」→ 回到 N04

POLL — 埋点轮询引擎（跳转节点）

节点类型跳转节点
切换规则按时间（每5秒触发一次）
静默时间5 秒
重复次数24 次（总计 120 秒等待窗口）
动作调用「查询埋点 API」，传入 exclusive_code
0–60s 无操作→ HINT 普通节点「看到了吗？」→ 回到 POLL
60–120s 无操作→ HINT 普通节点「短信直达链接」→ 回到 POLL
120s 超时→ 跳出循环 → 短信兜底（客户自闭环）
✅ 已点击链接→ 跳出循环 → N07

N01 — 意向筛选（LLM分类节点）

节点类型LLM分类节点
分类项有兴趣 / 犹豫想了解 / 明确不需要 / 已购买同类产品
指令（Prompt）根据用户回复判断其对医疗保障的兴趣。问细节/想了解→有兴趣；敷衍但没挂→犹豫；明确说不要→明确不需要；说已经买过→已购买
上下文轮数3
分类→有兴趣→ N02 信任验证
分类→犹豫→ N01-SOFT 话术软化 → N02
分类→拒绝→ 结束+短信（标记无意向）

N02 — 信任验证（LLM分类节点）

节点类型LLM分类节点
分类项信任通过 / 质疑但可解释 / 认为是诈骗
指令（Prompt）判断用户是否信任来电方。直接问保障内容→信任；问'哪家公司/怎么有我电话'→质疑但可解释；说'骗子/诈骗'→诈骗。诈骗分类直接转人工坐席
上下文轮数3
分类→信任→ N03 微信环境确认
分类→质疑可解释→ N02-TRUST 解释话术 → N03
分类→诈骗→ 🆘 转人工坐席（硬约束，不可被循环覆盖）

🆘 四条转人工阈值（AI→坐席）

  • 1
    客户反复质疑真实性 / 「是不是诈骗」 触发节点：N02 | 信任崩溃时AI无法自修复，必须人工建立信任
  • 2
    专属码连续输入错误 ≥ 3 次 触发节点：N08 | 系统侧或客户侧异常，AI无法诊断根因
  • 3
    健康告知拒保（智能核保不通过） 触发节点：N12 | 需人工推荐替代产品或解释核保结果
  • 4
    支付成功但核保系统异常 触发节点：N14 | 涉及资金安全，必须人工介入核保或退款

📱 短信兜底矩阵

  • N00 未接通 → 发送品牌名 + 公众号名 + 专属码
  • N01 无意向 → 发送公众号名 + 专属码（低打扰）
  • N03/N04 无微信/不愿关注 → 降级短信直达链接（跳过关注步骤）
  • N06 入口操作超时 120s → 挂机 + 直达投保页链接
  • N10 信息填写超时 → 提醒链接 24h 有效
  • N13 挂机未支付 → 30 分钟后支付链接 + 优惠过期提醒
  • N15 投保成功 → 投保成功确认 + 保单查询方式
