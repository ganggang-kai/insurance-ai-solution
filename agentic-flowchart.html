⚡ Skills 详细设计

Skill intent_screening — 建信与意向识别

触发时机外呼接通后，整个通话期间第一个被调用的 Skill
核心能力开场说明来意 → 自然对话建立信任 → LLM 语义判断客户意向（有兴趣/犹豫/拒绝/质疑） → 根据分类结果动态生成下一步话术
出口有意向 → 触发 wechat_guidance / 拒绝 → 结束通话 / 质疑 → LLM 自解释并重新评估

Skill wechat_guidance — 公众号引导

触发时机intent_screening 确认有意向后
核心能力LLM 引导客户搜索公众号 → 通过对话确认客户是否找到并关注 → 根据客户反应动态调整引导策略（已关注则跳过，未关注则自然引导，客户不会操作则分步讲解）
出口关注成功 → 触发 code_delivery

Skill code_delivery — 专属码管理

触发时机wechat_guidance 确认关注后
核心能力调用 Tool 获取专属码 → LLM 口播并确认客户收到 → 客户输入后校验 → 引导进入投保页
调用的 Toolget_exclusive_code（获取码）、query_tracking（校验码是否被使用）
出口码校验通过 → 触发 insurance_guidance

Skill insurance_guidance — 投保链路引导（双产品）

触发时机code_delivery 完成后，客户进入0.6元产品投保页
核心能力覆盖两个产品的投保引导全流程：
① 0.6元短险：引导进入投被保人页面 → 填写信息 → 点击《完善多重保障》
② 复购产品：引导点击《完善多重保障》（无需重新填写信息）
LLM根据 query_tracking 返回的埋点状态，动态判断客户所处环节并生成对应引导话术
调用的 Toolquery_tracking（查询页面进度、签约状态）
出口0.6元产品点击《完善多重保障》→ 触发 bankcard_sign / 复购产品点击《完善多重保障》→ 触发 bankcard_sign 或免签跳过

Skill bankcard_sign — 银行卡签约引导

触发时机客户点击《完善多重保障》后跳转至银行卡收银台
核心能力LLM分步引导银行卡签约全流程：选择银行卡 → 输入手机号验证码 → 银行卡签约授权 → 等待自动扣款。调用 query_tracking 实时查询签约状态，失败时分析原因并引导重试
调用的 Toolquery_tracking（查询签约页面状态及结果）
出口签约成功+扣款 → 过渡升级页（3s后自动跳转下一产品）

🔧 Tools 详细设计

Tool 名称对接客户接口功能描述调用 Skill
get_exclusive_code专属码获取接口为当前客户生成/获取四位专属码，返回码值及有效期code_delivery
query_tracking页面埋点查询接口查询客户在投保页的最新行为（页面进入、信息填写进度、银行卡签约页面状态等），覆盖两个产品全流程insurance_guidance bankcard_sign
query_underwriting核保结果查询接口（异步）提交基础资格校验请求并轮询结果，返回通过/不通过等状态（无健康告知，仅做基础资格判断）insurance_guidance
query_payment支付结果查询接口查询银行卡签约授权及扣款结果，返回成功/失败及失败原因bankcard_sign

🎯 选型场景

流程画布更适合合规要求极高、话术需逐条审批的场景；Agentic AI 更适合客户反应多变、追求更高转化率的场景。本方案基于我司 AI 外呼引擎对 Tool/Skill 的原生支持能力，提供 Agentic AI 的完整实现路径。

🧠 LLM 节点编排与 Prompt 设计

基于我司平台实际节点能力（LLM分类节点 + LLM回复节点），以下为每个 Skill 的具体节点编排方案及 Prompt 指令。

Skill 1 intent_screening — 建信与意向识别

步骤节点类型配置要点
1LLM回复节点 指令：
「你是保险服务顾问。客户刚接通电话，你需要：1）简短说明来电目的（提醒客户有一份专属医疗保障可领取）；2）询问客户是否有2分钟时间了解。保持自然亲切，不要推销，不要索要个人信息。如果客户表示忙，礼貌约定稍后联系。话术控制在100字以内。」
LLM回复最大次数：2次 | 上下文轮数：3 | 无效语义不打断：开
2LLM分类节点 分类项：有兴趣 / 犹豫想了解 / 明确不需要 / 质疑
指令：
「根据用户回复判断其对医疗保障的兴趣和信任状态：用户主动询问细节或表示想了解→有兴趣；用户敷衍但不挂断→犹豫；用户明确说不→明确不需要；用户问"你们是哪家公司/怎么有我电话/是不是骗子"→质疑。」
上下文轮数：5

Skill 2 wechat_guidance — 公众号引导

步骤节点类型配置要点
1LLM回复节点 指令：
「引导客户在微信中搜索公众号"XX保"并关注。说明关注后即可查看专属保障方案。如果客户表示找不到，分步指导：打开微信→顶部搜索框→输入"XX保"→选择蓝色认证账号→点关注。如果客户说已有/已关注，直接进入下一步。」
LLM回复最大次数：3次 | 上下文轮数：5
2判断节点 通过对话确认是否已关注。已关注 → code_delivery；引导2次仍不成功 → 记录后继续尝试

Skill 3 code_delivery — 专属码管理

步骤节点类型配置要点
1普通节点 调用 Tool: get_exclusive_code → 获取四位专属码
2LLM回复节点 指令：
「告知客户专属码（四位数字：{code}）。说明：在公众号首页输入这个码，即可自动打开保障页面。如果客户没记住，再重复一遍并提示"可以拿笔记一下"。不要催促。」
LLM回复最大次数：2次 | 上下文轮数：5

Skill 4 insurance_guidance — 投保链路引导（双产品）

步骤节点类型配置要点
1LLM回复节点 指令（0.6元产品—填写被保人信息）：
「客户已进入0.6元保障页面。引导客户查看保障内容，填写被保人姓名和身份证号。如果客户不知道身份证号，提示"一般自己的身份证号都能背下来，或者看一下身份证"。如果客户表示怕信息泄露，解释"这个页面是保险公司官方页面，数据加密，安全有保障"。填完后引导点击底部《完善多重保障》按钮。」
LLM回复最大次数：3次 | 上下文轮数：8 | 无效语义不打断：开
2跳转节点 每5秒调用 query_tracking 查询被保人信息填写进度。已填完→bankcard_sign；60秒无操作→LLM追问
3LLM回复节点 指令（复购产品）：
「客户已自动跳转到复购保障页面。告知客户"已为您自动升级到下一个保障产品"，引导点击《完善多重保障》按钮即可。无需重复填写信息。」
LLM回复最大次数：2次 | 上下文轮数：5

Skill 5 bankcard_sign — 银行卡签约引导

步骤节点类型配置要点
1LLM回复节点 指令：
「客户进入银行卡签约页面。分步引导：1）选择一张银行卡 2）输入手机号获取验证码 3）输入验证码 4）点击确认授权。每步确认客户完成后再引导下一步。如果客户卡在某步，针对性帮助（如"验证码在您手机的短信里"）。签约成功后告知"已自动扣款，正在为您升级保障"。如果失败，分析原因并引导重试。」
LLM回复最大次数：5次 | 上下文轮数：8 | 无效语义不打断：开
2跳转节点 每5秒调用 query_payment 查询签约及扣款结果。成功→升级过渡页；失败→回到LLM回复节点分析重试
3LLM分类节点 分类项（复购产品免签约判断）：免签约 / 需签约
指令：
「根据系统返回的复购产品签约模式判断：返回"skip_sign=true"→免签约；返回"skip_sign=false"→需签约。免签约直接告知完成，需签约走银行卡引导。」
上下文轮数：3

📊 节点使用汇总

节点类型数量用于
LLM回复节点5 个开场建信、公众号引导、专属码口播、投保页引导、银行卡签约引导
LLM分类节点2 个意向识别（有兴趣/犹豫/拒绝/质疑）、复购签约模式判断（免签/需签）
跳转节点2 个被保人信息填写进度轮询（query_tracking）、银行卡签约结果轮询（query_payment）
判断节点1 个关注状态确认（已关注/未关注）
普通节点1 个调用 get_exclusive_code 获取专属码
