⚡ Skills 详细设计

Skill intent_screening — 建信与意向识别

触发时机外呼接通后，整个通话期间第一个被调用的 Skill
核心能力开场说明来意 → 自然对话建立信任 → LLM 语义判断客户意向（有兴趣/犹豫/拒绝/质疑） → 根据分类结果动态生成下一步话术
调用的 Toolget_exclusive_code（如需提前生成专属码）
出口有意向 → 触发 wechat_guidance / 拒绝 → 结束通话 / 质疑 → LLM 自解释并重新评估

Skill wechat_guidance — 公众号引导

触发时机intent_screening 确认有意向后
核心能力LLM 引导客户搜索公众号 → 调用 Tool 查询关注状态 → 根据状态动态调整引导策略（已关注则跳过，未关注则自然引导，客户不会操作则分步讲解）
调用的 Toolquery_tracking
出口关注成功 → 触发 code_delivery

Skill code_delivery — 专属码管理

触发时机wechat_guidance 确认关注后
核心能力调用 Tool 获取专属码 → LLM 口播并确认客户收到 → 客户输入后校验 → 引导进入投保页
调用的 Toolget_exclusive_code（获取码）、query_tracking（校验码是否被使用）
出口码校验通过 → 触发 insurance_guidance

Skill insurance_guidance — 投保链路引导

触发时机code_delivery 完成后，客户进入投保页
核心能力LLM 全程陪伴式引导：引导入口点击 → 轮询埋点确认进度 → 引导信息填写 → 查询核保结果 → 根据结果动态沟通。LLM 可根据客户行为自主判断介入时机和话术内容
调用的 Toolquery_tracking（查询页面行为）、query_underwriting（查询核保结果，异步等待）
出口核保通过 → 触发 payment_followup

Skill payment_followup — 支付跟进

触发时机insurance_guidance 确认核保通过后
核心能力LLM 引导支付 → 调用 Tool 查询支付结果 → 根据失败原因（余额不足/银行卡受限/网络超时等）动态生成针对性引导话术 → 连续失败则暂停并记录
调用的 Toolquery_payment
出口支付成功 → 告知生效时间，结束通话

🔧 Tools 详细设计

Tool 名称对接客户接口功能描述调用 Skill
get_exclusive_code专属码获取接口为当前客户生成/获取四位专属码，返回码值及有效期code_delivery intent_screening
query_tracking页面埋点查询接口查询客户在公众号/投保页的最新行为（关注状态、页面点击、信息填写进度等）wechat_guidance code_delivery insurance_guidance
query_underwriting核保结果查询接口（异步）提交核保查询请求并轮询结果，返回通过/智能核保/拒保等状态insurance_guidance
query_payment支付结果查询接口查询当前客户的支付状态，返回成功/失败及失败原因payment_followup

🎯 选型场景

流程画布更适合合规要求极高、话术需逐条审批的场景；Agentic AI 更适合客户反应多变、追求更高转化率的场景。本方案基于我司 AI 外呼引擎对 Tool/Skill 的原生支持能力，提供 Agentic AI 的完整实现路径。
